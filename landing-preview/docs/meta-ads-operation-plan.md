# 파파컴퍼니 메타 광고 운영 기획안

## 1. 운영 목적

파파컴퍼니의 네이버 플레이스 상위노출 전략 문의를 확보하기 위한 메타 광고 운영 문서입니다. 초기 대상은 음식점 업종 사장님이며, 광고 목표는 아래 두 가지를 동시에 테스트합니다.

- 메타 안에서 바로 문의를 남기는 `문의 받기` 광고
- 랜딩페이지로 이동한 뒤 진단 신청을 남기는 `랜딩 문의` 광고

초기 성과 판단은 단순 문의 단가가 아니라 `유효 문의 비용`으로 봅니다. 유효 문의는 연락 가능한 음식점 사업자이며, 실제 네이버 플레이스 URL을 가지고 있고, 플레이스 노출 개선 니즈가 있는 문의를 의미합니다.

## 2. 캠페인 구조

| 구분 | 문의 받기 광고 | 랜딩 문의 광고 |
| --- | --- | --- |
| Objective | Leads | Leads |
| Conversion location | Instant form | Website |
| Performance goal | Maximize number of leads | Maximize number of conversions |
| 예산 비중 | 60% | 40% |
| 목적 | 빠른 리드 확보, 후킹 검증 | 설득된 고의도 리드 확보 |
| 저장 위치 | Google Sheet `시트2` | Google Sheet `시트` |
| CTA | 신청하기, 더 알아보기 | 더 알아보기, 무료 진단 |

운영 초기에는 문의 받기 광고로 리드 모수를 빠르게 확보하고, 랜딩 문의 광고로 리드 품질을 비교합니다. 최소 7일은 큰 구조 변경 없이 운영한 뒤, 유효 문의 비용 기준으로 예산을 재배분합니다.

## 3. 메타 광고 세팅값

### 캠페인 1. 문의 받기 광고

- Campaign objective: `Leads`
- Conversion location: `Instant form`
- Form type: `Higher intent`
- Performance goal: `Maximize number of leads`
- Budget: 전체 일 예산의 60%
- Placements: 초기에는 `Advantage+ placements`
- CTA: `신청하기` 또는 `더 알아보기`
- Lead form intro:
  - 제목: `음식점 플레이스 상위노출 전략 무료 진단`
  - 설명: `입력해주신 플레이스 URL을 기준으로 노출 개선 가능성을 확인한 뒤 영업일 기준 1~3일 이내 연락드립니다.`

### 캠페인 2. 랜딩 문의 광고

- Campaign objective: `Leads`
- Conversion location: `Website`
- Performance goal: `Maximize number of conversions`
- Destination: 랜딩페이지 상담 섹션 URL
- Budget: 전체 일 예산의 40%
- Placements: 초기에는 `Advantage+ placements`
- CTA: `더 알아보기` 또는 `무료 진단`

### 초기 예산

- 권장 일 예산: 30,000원~50,000원
- 30,000원 기준: 문의 받기 18,000원 / 랜딩 문의 12,000원
- 50,000원 기준: 문의 받기 30,000원 / 랜딩 문의 20,000원
- 첫 7일은 소재별 성과를 보기 위해 잦은 수정 금지

## 4. 상세 타겟팅 가이드

메타 Leads 캠페인은 Advantage+ Audience가 기본 적용되는 계정이 많습니다. 따라서 타겟을 과하게 좁히기보다, 음식점 사장님 신호를 충분히 주고 소재와 폼 질문으로 비고객을 걸러냅니다.

### 공통 Audience Controls

- Location: 한국
- Age: 28~55
- Gender: 전체
- Language: 한국어
- Placement: Advantage+ placements
- Exclusion:
  - 상담 완료 고객 리스트가 있으면 Custom Audience로 제외
  - 폼 제출 완료자 제외
  - 내부 직원/테스트 계정 제외 가능 시 제외

### Ad Set A. Broad Restaurant Owners

가장 넓은 음식점 사업자 후보군을 찾는 세트입니다.

- Audience type: Advantage+ Audience
- Audience suggestions:
  - 음식점
  - 식당
  - 외식업
  - 요식업
  - 자영업
  - 소상공인
  - 창업
  - 프랜차이즈
  - 배달의민족
  - 요기요
  - 네이버 지도
  - 네이버 플레이스
- 예산 비중: 신규 고객 예산의 50%
- 판단 기준: 유효 문의 수, 유효 문의 비용

### Ad Set B. Restaurant Operation Interest

매장 운영과 광고/마케팅 관심 신호가 있는 사람을 찾는 세트입니다.

- Audience type: Advantage+ Audience 또는 Original Audience
- Audience suggestions:
  - 음식점 경영
  - 레스토랑
  - 카페
  - 배달 음식
  - 포스 시스템
  - 예약
  - 리뷰
  - 로컬 비즈니스
  - 온라인 광고
  - 디지털 마케팅
  - 소셜 미디어 마케팅
- 예산 비중: 신규 고객 예산의 30%
- 판단 기준: 폼 제출률, 유효 문의율

### Ad Set C. Retargeting

데이터가 쌓인 뒤 2주차부터 분리합니다.

- 대상:
  - 랜딩 방문자 30일
  - 상담 섹션 도달자
  - 폼 오픈 후 미제출자
  - 인스타그램 계정 참여자 365일
  - 광고 영상 조회자 50% 이상
- 제외:
  - 폼 제출 완료자
  - 상담 완료 고객
- 예산 비중: 전체 예산의 20% 이내
- 목적: 망설인 사람에게 다시 무료 진단 제안

## 5. Instant Form 항목

문의 받기 광고의 Instant Form은 랜딩 폼과 동일하게 구성합니다.

| 항목 | 유형 | 필수 여부 | 비고 |
| --- | --- | --- | --- |
| 이름 | 기본 질문 | 필수 | 자동 입력 가능 |
| 전화번호 | 기본 질문 | 필수 | 자동 입력 가능 |
| 플레이스 URL | 단답형 | 필수 | 입력값 그대로 저장 |
| 메인 키워드 | 단답형 | 필수 | 예: 강남 맛집, 홍대 고기집 |
| 업종 | 단답형 또는 선택형 | 필수 | 음식점 세부 업종 |
| 대행사/직광고주 여부 | 선택형 | 필수 | 직광고주, 대행사 |
| 월 광고비 예산 | 선택형 | 필수 | 예산 구간 |
| 문의사항 | 장문형 | 필수 | 현재 고민 입력 |
| 개인정보 수집 및 이용 동의 | 동의 | 필수 | 개인정보처리방침 URL 연결 |
| 제3자 정보 제공 동의 | 동의 | 필수 | 필요 시 동의 문구 포함 |

폼 유형은 `Higher intent`로 설정해 제출 전 확인 단계를 둡니다. 문의 품질이 너무 낮으면 질문 수를 유지하고, 제출량이 지나치게 적으면 문의사항 항목만 선택 입력으로 완화하는 것을 2차 테스트로 검토합니다.

## 6. 시트2 적재 구조

메타 문의폼 리드는 Google Sheet의 `시트2`에 저장합니다. 랜딩 리드가 쌓이는 `시트`와 분리해 캠페인별 품질을 비교합니다.

| 컬럼 | 항목 | 비고 |
| --- | --- | --- |
| A | 접수일시 | Apps Script 저장 시점 |
| B | 메타 리드 ID | 중복 확인용 |
| C | 캠페인명 | Meta campaign name |
| D | 광고세트명 | Meta ad set name |
| E | 광고명 | Meta ad name |
| F | 이름 | 폼 입력값 |
| G | 전화번호 | 폼 입력값 |
| H | 플레이스 URL | 입력값 그대로 저장 |
| I | 플레이스 MID값 | URL에서 9~11자리 숫자 추출 |
| J | 수정된 플레이스 주소 | MID 기반 모바일 플레이스 URL |
| K | 메인 키워드 | 폼 입력값 |
| L | 업종 | 폼 입력값 |
| M | 대행사/직광고주 여부 | 폼 입력값 |
| N | 월 광고비 예산 | 폼 입력값 |
| O | 문의사항 | 폼 입력값 |
| P | 개인정보 동의 | 동의 여부 |
| Q | 제3자 제공 동의 | 동의 여부 |
| R | 후속 상태 | 미연락, 연락완료, 상담완료 등 |
| S | 상담 메모 | 운영자 기록 |

MID 수식 예시:

```excel
=IFERROR(REGEXEXTRACT(H2,"[0-9]{9,11}"),"")
```

수정된 플레이스 주소 수식 예시:

```excel
="https://m.place.naver.com/place/"&I2
```

## 7. Apps Script 연동 구조

Meta Instant Form은 일반 HTML 폼처럼 Apps Script URL로 바로 모든 값을 보내는 방식이 아닙니다. 직접 연동 시 구조는 아래처럼 잡습니다.

```text
Meta Lead Webhook
→ Apps Script doPost
→ Graph API로 leadgen_id 상세 조회
→ 응답 필드 매핑
→ Google Sheet 시트2 저장
```

필요 설정:

- Meta Developer App 생성
- Page leadgen webhook 연결
- Webhook verify token 설정
- Page access token 발급 및 보관
- leadgen_id로 리드 상세 조회
- Apps Script Web App 배포
- 중복 lead ID 저장 방지

빠른 임시 운영이 필요하면 Make/Zapier를 사용해 `Meta Lead Ads → Google Sheets 시트2`로 먼저 연결하고, 직접 Webhook은 2차 개발로 전환할 수 있습니다.

## 8. 원페이지 소재 제작 가이드

소재는 카드뉴스가 아니라 한 장에서 메시지가 끝나는 원페이지 이미지로 제작합니다.

### 공통 레이아웃

- 상단 20%: 강한 후킹 헤드라인
- 중앙 45%: 문제 상황 또는 사례형 비주얼
- 하단 25%: 진단 항목 3개와 CTA
- 최하단 10%: 파파컴퍼니 로고와 무료 진단 문구

### 디자인 규칙

- 랜딩과 같은 보라 포인트 컬러 사용
- 배경은 연한 보라/화이트 계열
- 핵심 문장은 검정 또는 아주 진한 남보라
- CTA는 보라색 버튼 형태
- 한 소재의 핵심 문장은 최대 2개
- 모바일에서 1초 안에 읽히도록 헤드라인을 크게 배치
- 작은 설명문을 많이 넣지 않음
- `순위 보장`, `무조건 1위`, `매출 보장` 표현 금지

### 권장 사이즈

- 1:1 피드: 1080 x 1080
- 4:5 피드: 1080 x 1350
- 9:16 스토리/릴스: 1080 x 1920

## 9. 후킹 문구 리스트

### 손실 자극형

- `근처 손님을 경쟁 매장에 뺏기고 있을 수 있습니다`
- `광고비 쓰기 전에 플레이스부터 망가져 있을 수 있습니다`
- `손님은 검색하는데 우리 가게는 뒤에서 멈춰 있을 수 있습니다`
- `검색 노출이 약하면 좋은 메뉴도 발견되지 않습니다`

### 경쟁 비교형

- `같은 골목인데 왜 옆 가게만 먼저 보일까요?`
- `리뷰가 있어도 경쟁 매장에 밀릴 수 있습니다`
- `맛집 검색 결과에서 뒤로 밀리는 매장에는 이유가 있습니다`
- `상위 매장과 하위 매장은 보여주는 정보부터 다릅니다`

### 광고비 방어형

- `플레이스가 정리되지 않으면 광고비가 새어 나갑니다`
- `검색 노출이 약한 상태에서 광고만 키우면 손해입니다`
- `광고보다 먼저 봐야 할 건 플레이스 상태입니다`
- `광고 클릭 전에 고객은 이미 플레이스를 보고 판단합니다`

### 사례 기대형

- `플레이스 순위가 바뀌면 문의 흐름도 달라집니다`
- `노출이 막힌 지점을 찾으면 개선 방향이 보입니다`
- `상위노출 전략은 매장 상태를 보는 것부터 시작됩니다`
- `우리 매장에 맞는 플레이스 개선 방향을 받아보세요`

## 10. 소재별 제작 브리프

### 소재 A. 경쟁 매장 비교형

- 헤드라인: `같은 골목인데 왜 옆 가게만 먼저 보일까요?`
- 보조문구: `플레이스 노출 구조부터 무료로 진단합니다`
- 비주얼:
  - 두 개의 검색 결과 카드 배치
  - 경쟁 매장은 상단에 선명하게 표시
  - 우리 매장은 아래쪽에 흐릿하게 표시
- CTA: `무료 진단 신청`
- 목적: 음식점 사장님의 경쟁 불안을 자극

### 소재 B. 손실 자극형

- 헤드라인: `근처 손님을 경쟁 매장에 뺏기고 있을 수 있습니다`
- 보조문구: `네이버 플레이스 상위노출 전략을 확인하세요`
- 비주얼:
  - 지도/검색 화면 느낌
  - 손님 흐름이 경쟁 매장 쪽으로 이동하는 구조
  - 우리 매장은 화면 아래에 작게 노출
- CTA: `상위노출 전략 받기`
- 목적: 지금 놓치고 있는 수요를 인식시키기

### 소재 C. 광고비 방어형

- 헤드라인: `광고비 쓰기 전에 플레이스부터 확인하세요`
- 보조문구: `노출을 막는 항목을 먼저 찾아드립니다`
- 비주얼:
  - 광고비 아이콘
  - 플레이스 체크리스트
  - 빨간 경고 요소는 최소화하고 보라 체크 표시 사용
- CTA: `무료 진단 받기`
- 목적: 광고비를 쓰는 사장님에게 예산 누수 문제 제기

### 소재 D. 리뷰 역설형

- 헤드라인: `리뷰가 있어도 경쟁 매장에 밀릴 수 있습니다`
- 보조문구: `사진, 메뉴, 키워드, 운영 상태까지 함께 봅니다`
- 비주얼:
  - 별점/리뷰 수는 높은데 검색 위치는 낮은 대비
  - 상단 매장과 하단 매장의 정보량 비교
- CTA: `내 매장 진단하기`
- 목적: 리뷰가 전부가 아니라는 인식 전환

### 소재 E. 상위노출 전략형

- 헤드라인: `음식점 플레이스 상위노출, 전략 없이 어렵습니다`
- 보조문구: `우리 매장에 맞는 노출 개선 방향을 제안합니다`
- 비주얼:
  - 순위 상승 그래프
  - 체크된 플레이스 항목 3개
  - 보라색 상승 화살표
- CTA: `전략 제안 받기`
- 목적: 무료 진단의 결과물이 전략 제안이라는 점 강조

## 11. 성과 판단 기준

### 주요 지표

- 문의 수
- CPL
- 폼 오픈 수
- 폼 제출률
- 랜딩 클릭률
- 랜딩 문의 전환율
- 유효 문의율
- 유효 문의 비용
- 연락 성공률
- 상담 전환율

### 유효 문의 조건

- 전화번호가 실제 연결 가능함
- 음식점 업종임
- 플레이스 URL이 정상 입력됨
- 노출 개선 또는 상위노출 니즈가 있음
- 상담 연결 가능성이 있음

## 12. 7일 운영 후 의사결정 룰

7일차에는 광고 관리자 수치와 시트 기록을 함께 봅니다.

- CPL은 낮지만 유효 문의율이 낮으면 예산 축소
- CPL은 높아도 상담 전환율이 높으면 유지 또는 증액
- CTR은 높은데 폼 제출률이 낮으면 폼 첫 화면 문구와 질문 순서 수정
- 폼 오픈은 많은데 제출이 적으면 `Higher intent` 유지 여부 검토
- 랜딩 클릭은 많은데 문의가 적으면 랜딩 상담 섹션 도달성과 CTA 위치 점검
- 특정 후킹 소재에서 유효 문의가 나오면 4:5와 9:16으로 확장
- 7일간 유효 문의가 전혀 없으면 타겟보다 오퍼와 소재 후킹을 먼저 수정

## 13. 금지 문구 및 주의사항

아래 방향은 사용하지 않습니다.

- 순위 또는 매출을 보장하는 표현
- 너무 일반적인 점검형 문구
- 네이버 공식 제휴처럼 오해될 수 있는 표현
- 특정 경쟁 매장을 직접 비방하는 표현
- 사용자가 삭제 요청한 약한 후킹 문구

광고 심사와 신뢰도를 위해 “보장”이 아니라 “진단”, “전략 제안”, “개선 방향 확인” 중심으로 표현합니다.
