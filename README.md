# 파파인사이트

파파인사이트는 파파컴퍼니 ERP 홈페이지입니다. 현재는 1차 `MVP`(엠브이피: 최소 기능 제품) 단계이며, 직원 로그인 후 내부 대시보드를 사용하는 구조입니다.

## 로컬 실행

```powershell
node dev-server.js
```

브라우저 접속 주소:

```text
http://localhost:4175/
```

테스트 계정:

```text
아이디: papa.admin
비밀번호: 1234
```

관리자 아이디는 `admin`, `관리자`로도 로그인할 수 있습니다.

## 현재 기능

- 직원 로그인
- 영업자료 (이미지)
  - 카테고리별 이미지 등록
  - 이미지 다운로드/복사
  - 등록자 표시
  - 삭제 처리 및 삭제 기록
  - 카테고리 접기/펼치기
  - 이미지 카드 펼쳐보기
- 영업자료 (텍스트)
  - 기존 파파AI 자료를 텍스트 영업자료 메뉴로 재구성
  - PAPAchatbot 문서 2개를 표 중심으로 표시
- 거래처 관리
  - 담당자: 이신, 정완, 현민, 오찬, 도영
  - B2C / B2B 구분
  - 거래중 / 상담중 / 대기 상태 구분
- 정산관리
  - 준비중 화면 표시

## 운영 주의사항

- 현재 영업자료 이미지 등록/삭제 기록은 브라우저 `localStorage`(로컬스토리지: 브라우저 내부 저장소)에 저장됩니다.
- 여러 직원이 같은 자료를 공유하려면 백엔드와 `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스)가 필요합니다.
- `GitHub`(깃허브: 코드 저장소 서비스) 저장, `commit`(커밋: 변경 내용을 저장소 기록으로 확정), `push`(푸시: 원격 저장소에 업로드)는 사용자 승인 후 진행합니다.

## 2026-06-01 영업자료 3분류 보관함

- `영업자료 (이미지)` 화면의 상단 보관함을 3개로 분리했습니다.
- 분류는 `단가 및 설명 이미지`, `레퍼런스 이미지`, `레퍼런스 링크`입니다.
- `레퍼런스 링크`는 여러 URL을 한 번에 입력해도 링크 1개당 카드 1개가 아니라, 하나의 묶음 카드로 저장됩니다.
- 링크 묶음 카드의 복사 기능은 묶음 안의 전체 URL을 줄바꿈 형태로 복사합니다.
- 기존 `레퍼런스 이미지 및 링크` 자료 중 이미지가 있는 자료는 `레퍼런스 이미지`, 링크만 있는 자료는 `레퍼런스 링크`로 자동 분류됩니다.
## 2026-06-01 공유 영업자료 저장

- 로컬 서버(`dev-server.js`)는 `/api/materials` `API`(에이피아이: 프로그램끼리 데이터를 주고받는 연결 방식)를 제공한다.
- 이 `API`는 영업자료 이미지 등록, 삭제, 삭제 로그를 `data/materials-state.json`에 저장한다.
- `data/materials-state.json`은 PC별 테스트/운영 데이터라 `GitHub`(깃허브: 코드 저장소 서비스)에 올리지 않는다.
- 운영 도메인에서 여러 PC가 같은 자료를 보려면 정적 파일 배포만으로는 부족하며, `/api/materials`를 제공하는 백엔드 또는 `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스) 연결이 필요하다.

## 2026-06-01 영업자료 보관함 구조

- `영업자료 (이미지)` 화면은 `단가 및 설명 이미지`, `레퍼런스 이미지 및 링크` 두 보관함으로 나뉜다.
- 각 보관함에는 별도 `자료 추가` 버튼이 있으며, 등록 시 같은 상품 카테고리 목록을 사용한다.
- 레퍼런스 보관함은 이미지 파일 또는 URL(유알엘: 웹 링크 주소)을 등록할 수 있고, 링크만 등록한 자료도 카드로 표시된다.
- 상단 보관함 카드를 클릭하면 선택한 보관함의 카테고리 리스트만 표시된다.
- 공통 카테고리에는 `클립 상위노출`이 포함된다.
- 등록된 자료는 카드의 `수정` 버튼으로 이름, 카테고리, 링크, 이미지 내용을 바꿀 수 있다.
- 카드 체크박스로 원하는 자료를 선택한 뒤 전체 선택, 선택 다운로드, 선택 복사를 사용할 수 있다.
- 카테고리별 자료 수는 우측 배지 카드로 표시되며, 카테고리별 전체 선택도 지원한다.
- 이미지 미리보기 클릭은 선택 토글, `펼쳐보기` 버튼은 확장 보기로 동작한다.
- 삭제 기록은 삭제자 계정별로 묶어서 표시된다.
- 레퍼런스 링크는 여러 줄로 한 번에 등록할 수 있다.
- 영업자료 화면에는 카테고리 검색과 자동완성 후보가 제공된다.

## 2026-06-05 영업자료 화면 최신 작업 상태

- 영업자료(이미지)는 대분류, 중분류, 소분류 카드 구조로 관리합니다.
- 소분류 검색은 현재 대분류 안에서 맞는 중분류와 소분류로 자동 이동합니다.
- 선택 소분류 영역에는 전체 선택, 선택 해제, 개별 체크 선택 기능이 있습니다.
- 영업자료(텍스트)는 기존 텍스트 자료 데이터(`리워드(트래픽)`, `블로그`, `영수증`)를 유지하고, 화면 미장과 카드 구조만 영업자료(이미지)와 비슷하게 정리했습니다.
- 로컬 실행 주소는 `http://localhost:4175/`입니다.
- 검증: `node --check script.js`, 로컬 서버 200 OK, 검색 로직 케이스 검증을 완료했습니다.
