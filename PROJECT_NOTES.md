- 프로젝트명: 파파인사이트
- 성격: 파파컴퍼니 ERP(회사 내부 업무 관리 시스템) MVP(엠브이피: 최소 기능 제품)

- 거래처 관리

## 작업 운영 결정

- `GitHub`(깃허브: 코드 저장소 서비스) 저장, `commit`(커밋: 변경 내용을 저장소 기록으로 확정), `push`(푸시: 원격 저장소에 업로드)는 사용자 승인 없이 진행하지 않는다.
- 오류나 반복된 실수는 `ERROR_LOG.md`에 기록한다.
- 화면 작업 후에는 브라우저에서 직접 확인하고 가능하면 `E2E`(이투이: 사용자 흐름 전체 검증)를 진행한다.
- 현재 이미지 등록/삭제 기록은 브라우저 `localStorage`(로컬스토리지: 브라우저 내부 저장소)에 저장된다.
- 여러 PC와 직원이 같은 영업자료를 공유하려면 추후 백엔드와 `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스)가 필요하다.

## 2026-06-01 영업자료 이미지 공유 저장 결정

- 영업자료 이미지 등록/삭제 데이터는 더 이상 각 PC의 `localStorage`(로컬스토리지: 해당 브라우저 안 저장소)에만 의존하지 않고, 로컬 서버의 `/api/materials` `API`(에이피아이: 프로그램끼리 데이터를 주고받는 연결 방식)를 우선 사용한다.
- `/api/materials`는 `data/materials-state.json`에 등록 이미지, 삭제된 기본 이미지 ID, 삭제 로그를 함께 저장한다.
- 서버 `API`가 없는 정적 배포 환경에서는 기존처럼 `localStorage`로만 동작하므로, 여러 PC가 같은 자료를 공유하려면 운영 도메인에서도 해당 `API` 또는 별도 `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스) 연결이 필요하다.
- 자료 추가 팝업은 1장, 2장, 3장 이상 다중 이미지 등록을 지원하며, 한 번 등록한 뒤 다시 등록해도 버튼 상태가 복구되도록 오류 처리와 등록 중 버튼 비활성화를 적용한다.

## 2026-06-01 영업자료 보관함 분리

- 영업자료 이미지 화면 상단의 기존 `영업자료 이미지 보관함`을 `단가 및 설명 이미지`와 `레퍼런스 이미지 및 링크` 두 보관함 카드로 분리했다.
- `단가 및 설명 이미지`는 기존 단가표/설명 이미지 카테고리를 관리하는 보관함이며, 기존 기본 이미지와 등록 이미지의 기본 분류는 이 보관함으로 유지한다.
- `레퍼런스 이미지 및 링크`는 상품별 참고 이미지 또는 외부 링크를 등록하는 보관함이다. 링크만 등록하는 자료도 카드로 표시한다.
- 전체 배경과 주요 카드 스타일은 블랙/그레이 기반에 초록/민트 네온 포인트가 들어가는 방향으로 조정했다.
- 두 보관함 상단 카드는 탭처럼 동작한다. `단가 및 설명 이미지`를 클릭하면 해당 보관함 리스트만 보이고, `레퍼런스 이미지 및 링크`를 클릭하면 레퍼런스 리스트만 보인다.
- 공통 카테고리에 `클립 상위노출`을 추가해 양쪽 보관함에서 동일하게 등록할 수 있게 했다.
- 영업자료 카드는 수정 기능을 제공한다. 이름, 카테고리, 링크, 이미지 파일을 수정할 수 있으며, 수정 시 새 이미지를 선택하지 않으면 기존 이미지가 유지된다.
- 영업자료 카드에는 체크박스가 있으며 현재 보관함 기준 전체 선택, 선택 다운로드, 선택 복사를 지원한다.
- 이미지 목록은 한눈에 더 많이 보이도록 촘촘한 카드 그리드로 조정하고, 이미지 클릭 자체로 카테고리 목록이 접히지 않도록 이벤트 흐름을 분리했다.
- 카테고리별 자료 수는 우측 네온 배지 카드로 표시하고, 카테고리별 전체 선택 버튼을 제공한다.
- 보관함+카테고리 단위의 펼침 상태를 기억해 체크/이미지 선택/전체 선택 후에도 사용자가 보고 있던 카테고리가 접히지 않도록 유지한다.
- 삭제 기록은 삭제자 계정명 기준 그룹으로 표시한다. 삭제 기록 데이터는 `/api/materials` 공유 상태의 `deleteLogs`에 저장되므로 다른 PC에서도 같은 기준으로 확인된다.
- 레퍼런스 보관함 링크 등록은 여러 줄 입력을 지원한다. 한 줄에 하나씩 URL(유알엘: 웹 링크 주소)을 입력하면 링크 자료 카드가 여러 개 생성된다.
- 영업자료 카테고리 검색은 자동완성 후보와 함께 제공하며, 입력 키워드가 포함된 카테고리만 보여준다.
- 영업자료 외 인바운드 DB, 영업자료 텍스트, 거래처 관리 화면도 블랙/그레이 기반 네온 카드 톤으로 맞춘다.
## 2026-06-01 영업자료 3분류 보관함 결정

- 기존 `레퍼런스 이미지 및 링크` 보관함은 `레퍼런스 이미지`와 `레퍼런스 링크`로 분리한다.
- 최종 보관함 구조는 `단가 및 설명 이미지`, `레퍼런스 이미지`, `레퍼런스 링크` 3개다.
- `레퍼런스 링크`는 여러 URL을 한 번에 입력할 수 있으며, URL마다 카드를 나누지 않고 하나의 링크 묶음 카드로 저장한다.
- 기존 `reference` 자료는 이미지가 있으면 `reference-image`, 이미지 없이 링크만 있으면 `reference-link`로 자동 분류한다.
- `레퍼런스 링크` 카드 복사 기능은 카드 안의 전체 링크를 줄바꿈 목록으로 복사한다.
