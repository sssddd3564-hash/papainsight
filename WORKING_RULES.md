# 작업 운영 규칙

파파인사이트 작업을 이어갈 때 적용할 운영 규칙입니다.

## 문서 우선

- 작업 전 `AGENTS.md`, `PROJECT_NOTES.md`, `CHANGELOG.md`, `README.md`를 확인합니다.
- 작업 중 변경된 기획, 구조, 기능, 결정사항은 관련 `MD`(엠디: Markdown 문서)에 기록합니다.
- 컨텍스트(콘텍스트: 대화/작업 기억)가 길어질 때는 압축 전에 문서를 최신화합니다.

## 검증

- 화면 작업은 오른쪽 브라우저에서 직접 확인합니다.
- 가능하면 `E2E`(이투이: 사용자 흐름 전체 검증)를 진행합니다.
- 데스크톱, 태블릿, 모바일 화면에서 주요 UI가 깨지지 않는지 확인합니다.
- `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스), 백엔드, 결제, 관리자 기능은 필요한 정보 없이 임의 검증하지 않습니다.

## 오류 기록

- 실패한 명령, 반복된 실수, 잘못된 추정, 검증 중 발견한 문제는 `ERROR_LOG.md`에 기록합니다.
- 수정 후에는 원인과 해결 내용을 함께 남깁니다.

## 저장과 배포

- `GitHub`(깃허브: 코드 저장소 서비스) 저장, `commit`(커밋: 변경 내용을 저장소 기록으로 확정), `push`(푸시: 원격 저장소에 업로드)는 사용자 승인 후 진행합니다.
- 기능과 `DB` 관련 작업은 10분 단위로 백업하거나 최소한 문서와 변경 이력에 저장합니다.
- 운영 도메인 배포 전에는 캐시, 권한, 공개 범위, 로그인 정보를 함께 확인합니다.

## 보안

- 비밀번호, 토큰, 개인정보, 관리자 계정 정보가 코드에 노출되지 않도록 확인합니다.
- 현재 정적 MVP에는 직원 로그인 정보가 프론트 코드에 들어가 있으므로 실제 운영 전에는 백엔드 인증으로 전환해야 합니다.
