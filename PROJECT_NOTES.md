# 프로젝트 노트

이 문서는 파파인사이트의 확정된 작업 맥락과 운영 결정을 기록합니다.

## 현재 프로젝트

- 프로젝트명: 파파인사이트
- 성격: 파파컴퍼니 ERP(회사 내부 업무 관리 시스템) MVP(엠브이피: 최소 기능 제품)
- 기본 구조: 정적 `HTML`(에이치티엠엘: 화면 구조 코드), `CSS`(씨에스에스: 화면 스타일 코드), `JavaScript`(자바스크립트: 화면 동작 코드)
- 로컬 실행 주소: `http://localhost:4175/`
- 운영 도메인: `https://papainsight.co.kr/`

## 현재 주요 기능

- 직원 로그인 화면
- 영업자료 (이미지)
  - 카테고리별 이미지 등록
  - 이미지 다운로드/복사
  - 이미지 삭제
  - 삭제 기록 저장
  - 등록자 표시
  - 카테고리 접기/펼치기
  - 이미지 카드 펼쳐보기
- 영업자료 (텍스트)
  - 기존 파파AI 메뉴를 영업자료 문서 보관함으로 변경
  - `PAPAchatbot` 저장소에서 가져온 핵심 문서 2개를 표 중심으로 표시
- 거래처 관리
  - 담당자: 이신, 정완, 현민, 오찬, 도영
  - 담당자별 하위 메뉴 접기/펼치기
  - B2C / B2B 분리
  - 거래중 / 상담중 / 대기 상태 분리
- 정산관리
  - 현재 준비중 화면으로 표시

## 작업 운영 결정

- `GitHub`(깃허브: 코드 저장소 서비스) 저장, `commit`(커밋: 변경 내용을 저장소 기록으로 확정), `push`(푸시: 원격 저장소에 업로드)는 사용자 승인 없이 진행하지 않는다.
- 오류나 반복된 실수는 `ERROR_LOG.md`에 기록한다.
- 화면 작업 후에는 브라우저에서 직접 확인하고 가능하면 `E2E`(이투이: 사용자 흐름 전체 검증)를 진행한다.
- 현재 이미지 등록/삭제 기록은 브라우저 `localStorage`(로컬스토리지: 브라우저 내부 저장소)에 저장된다.
- 여러 PC와 직원이 같은 영업자료를 공유하려면 추후 백엔드와 `DB`(디비: 데이터를 저장하고 관리하는 데이터베이스)가 필요하다.

## 2026-05-28 v0.1 확정 사항

- 현재 최신 화면 버전은 `v0.1`로 표기한다.
- 로그인 페이지에는 테스트 계정 안내를 노출하지 않고, 버전 정보만 작게 표시한다.
- 메인 대시보드에서는 기존 `MVP Console` 문구 위치에 `v0.1`을 표시한다.
- 모든 주요 대시보드 상단에는 현재 버전을 확인할 수 있는 버전 배지를 둔다.
- 영업자료 이미지 카드는 기본적으로 촘촘하게 보이게 유지하고, 사용자가 펼쳐보기를 누르면 느린 확장 모션으로 이미지를 크게 확인할 수 있게 한다.
- 거래처 관리 담당자 하위 메뉴는 담당자 클릭 시 접히지 않고 열린 상태를 유지한다.
- 오른쪽 브라우저 확인용 상단 툴바는 `파파인사이트 프로젝트` 제목, 섹션 전환(`내부 ERP`, `랜딩페이지`), 화면 크기 전환(`데스크톱`, `태블릿`, `모바일`, 직접 px 입력)을 포함한다.
- 랜딩페이지는 `lcm67088-tech/B2CRanding` GitHub 최신본을 `landing-preview/` 폴더로 내려받아 iframe(아이프레임: 다른 웹페이지를 현재 화면 안에 넣는 HTML 요소)으로 표시한다.
- 현재 랜딩페이지 미리보기 연결 파일은 `landing-preview/index-v2.html`이며, 이 파일은 `styles-v2.css`, `script-v2.js`를 사용한다.
- `landing.papainsight.co.kr` 운영 서버에는 `B2CRanding` 최신 v2 작업물을 배포했다. 서버에서는 `index-v2.html`을 운영용 `index.html`로 배치하고, `styles-v2.css`, `script-v2.js`, `assets/`를 함께 배포한다.
