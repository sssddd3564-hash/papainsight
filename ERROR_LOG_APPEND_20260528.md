# 추가 오류 기록 - 2026-05-28

## GitHub 최신본 반영 후 반응형 미리보기 툴바 누락

문제:

`AGENTS.md`에는 오른쪽 브라우저 확인 시 크기 조절/반응형 미리보기 툴바를 적용하라는 규칙이 있었지만, GitHub 최신본을 로컬에 덮어쓴 뒤 실제 `index.html`, `styles.css`, `script.js`에는 툴바 구현이 없었다.

원인:

이전 로컬 작업에서 추가했던 툴바 구현이 GitHub 최신본에는 포함되어 있지 않았고, 최신본 반영 과정에서 해당 로컬 구현이 사라졌다.

해결:

최신본 위에 상단 크기 지정 바, 오른쪽 반응형 미리보기 툴바, 크기 조절 스크립트를 다시 추가했다. 캐시 방지를 위해 `styles.css`와 `script.js` 버전 쿼리도 `0.1-preview-toolbar-20260528`로 변경했다.

다음부터:

GitHub 최신본을 가져온 뒤에는 문서 규칙만 확인하지 말고, 규칙에 필요한 실제 구현이 파일에 남아 있는지 확인한다.

## 상단 전환형 미리보기 툴바 요구사항 구체화

문제:

기존 구현은 오른쪽 세로 툴바 중심이었지만, 사용자는 이미지 예시처럼 상단 가로 툴바에서 내부 ERP와 랜딩페이지를 전환하며 작업하기를 원했다.

해결:

상단 툴바를 `파파인사이트 프로젝트` 기준으로 재구성하고, 섹션 버튼(`내부 ERP`, `랜딩페이지`)과 화면 크기 버튼(`데스크톱`, `태블릿`, `모바일`, px 입력, 적용, 맞춤 보기)을 추가했다. 랜딩페이지는 iframe으로 같은 화면 안에서 전환되도록 했다.

다음부터:

"브라우저 확인 툴바" 요청은 단순 반응형 크기 조절만이 아니라, 관련 작업물 간 전환 UI까지 포함할 수 있는지 먼저 확인한다.
